## FEATURE 1: HayatAğı (Life Web)
This is the CORE daily-use feature that makes users open the 
app every single day.

Every evening at 21:00 local time, send a push notification: 
"İyisin? / Are you OK?"

User sees a single screen with two large buttons:
- ✓ İyiyim (I'm OK) — green
- 🆘 Yardım (Help) — red

If user does NOT respond within 15 minutes:
- Send automatic alert to all emergency contacts via in-app 
  notification
- Show alert on map: "[Name] has not confirmed status 
  at [location]"
- If internet is unavailable, queue the alert for BLE mesh relay

HayatAğı Network View:
- Show a visual "web" of connected contacts
- Each node = a person
- Green = confirmed OK today
- Yellow = not yet responded
- Red = alert triggered
- Animated connections between nodes
- Users can see their whole family/friend network status at a glance

Group feature: Create a HayatAğı group (family, class, 
neighborhood). All members see each other's daily status.
Group admin can set custom check-in times.

Statistics: "Your network has been safe for 47 consecutive days" 
Streak system, gentle gamification.

## FEATURE 2: BLE Deneyap Kart Connection
- Bluetooth Low Energy scanner
- Scan for nearby ORBIT-MESH nodes (advertise UUID: 
  "ORBIT-MESH-NODE")
- On connection: display node ID, battery level, signal strength
- Real-time data stream from node:
  * Magnetometer readings (x, y, z in µT)
  * VLF signal strength (0-100 scale)
  * Node uptime
  * Mesh network: how many other nodes detected
- Send commands to node: 
  * "SEND_TEST_MSG:[message]" 
  * "SET_ALERT_MODE:1/0"
  * "REQUEST_STATUS"
- Show live graph of magnetometer data (last 60 seconds)
- Connection log with timestamps

## FEATURE 3: Phone Magnetometer Monitor
- Access device magnetometer via Web API 
  (if available on device)
- Show real-time 3-axis readings with animated gauge
- Live chart: rolling 5-minute history
- Baseline calibration: user sets "normal" for their location
- Anomaly detection: if reading deviates >15% from baseline 
  for >30 seconds, trigger soft alert
- "Cosmic Event" detector: compare with NASA DONKI API 
  for active geomagnetic storms
  (API: https://api.nasa.gov/DONKI/GST?api_key=DEMO_KEY)
- Manual log: user can mark and save anomaly events 
  with notes
- Export data as CSV

  * Log celestial events: what, when, location, description, photo
  * Each log gets GPS coordinates auto-filled
  * Privacy: public or private
  
- Community Feed: 
  * Browse public observations from other users on a map
  * Filter by: event type, date, location radius

  * Leaderboard by city and country
  * Badge system: Bronze/Silver/Gold observer

## FEATURE 5: Nabız (Pulse) — THE 2 BILLION USER FEATURE
This is the feature that makes ORBIT-MESH irreplaceable globally.

Concept: A passive, continuous "heartbeat" of your network.
Unlike HayatAğı (daily check-in), Nabız is always-on background 
monitoring.

How it works:
- Every user's app passively logs: 
  * "Last app open" timestamp
  * Phone magnetometer reading (once per hour, background)
  * Approximate location (city-level only, privacy-first)
  
- Your Nabız score: a simple number 1-100 showing 
  "network health" — how recently all your contacts were active

- If ANY contact goes silent for user-defined period 
  (default: 48 hours), gentle notification: 
  "[Name] hasn't been seen in 2 days. Check in?"

- During disasters: Nabız switches to high-frequency mode 
  (every 15 min check-in)

- Global Nabız Map: 
  * World map showing aggregated "pulse" density
  * Bright areas = active users
  * Dimming areas = possible communication disruption
  * This map becomes a real-time disaster detection tool
  * If a city's Nabız suddenly drops: possible disaster indicator
  * Completely anonymous at global level

- "Dünya'nın Nabzı" (World's Pulse) — public dashboard:
  * Show live global activity heat map
  * Historical: "Istanbul's pulse dropped 73% on Feb 6, 2023 
    at 04:17" — earthquake correlation visualization
  * Makes the app feel connected to something bigger than yourself

Why this gets 2 billion users:
- Grandparents use it to feel connected to grandchildren
- Parents use it for kids at university
- Friends use it to casually stay connected without 
  messaging pressure
- Scientists use it for disaster correlation research
- It costs nothing, requires nothing — just have the app

## TECHNICAL REQUIREMENTS

Backend (Node.js + Express):
- /api/auth: register, login, logout, me
- /api/hayat-agi: get network, update status, create group
- /api/observations: CRUD for astronomy logs

  nabiz_logs, missions, mission_completions

   - Top: "Dünya'nın Nabzı" mini widget
   - HayatAğı network circle (animated)
   - Today's check-in status for all contacts
   - Quick astronomy widget (tonight's sky)
   - Active mission banner

3. HayatAğı page:
   - Full network visualization (D3.js force graph or 
     simple CSS circles)
   - Add/remove contacts
   - Group management
   - Check-in history calendar

4. Magnetometer page:
   - Live gauges (BLE device OR phone sensor)
   - 5-minute chart
   - Anomaly log
   - Cosmic event correlation panel

5. Astronomy page:
   - Map of community observations

   - My network health score
   - Global pulse statistics

7. Profile/Settings page

Design details:
