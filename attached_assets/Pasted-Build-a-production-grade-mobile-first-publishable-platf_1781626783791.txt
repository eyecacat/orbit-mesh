Build a production-grade, mobile-first, publishable platform named ORBIT-MESH.

This is a real product, not a demo, not a mockup, not a simulation, and not a placeholder prototype.

CORE MISSION
Create a scalable, real, user-facing application that unifies:
- HELIO / VLF astronomy observation
- BLE-connected Deneyap Kart nodes
- Mesh-based message and sensor relay
- EarthSign authenticity layer
- AUET acoustic/underground event tracking
- HayatAğı family/safety network
- Astronomy gamification
- 50-question competitive astronomy quiz with global leaderboard
- Real user auth and profiles
- Real 3D UI/UX interactions

PRIMARY REQUIREMENTS
1) Real BLE integration
- BLE must work with actual hardware.
- Use a real BLE implementation, not simulated data.
- If BLE is unavailable, show a blocked / disconnected state. Do not fake device data.
- Support scanning, connecting, reconnecting, parsing JSON payloads, and live sensor streaming.

2) Real authentication and user system
- Email/password registration and login
- JWT/session auth
- User profiles
- Saved devices/nodes
- Family/group management
- Emergency contacts
- Activity history
- Leaderboard profile

3) Real backend
- Persistent database
- Real API endpoints
- Sensor logs
- Quiz results
- Leaderboard rankings
- Device registry
- User groups
- Notification records
- Event history
- Audit-friendly data model

4) Real-time device + app integration
- Connect to Deneyap Kart nodes over BLE
- Parse live JSON payloads
- Maintain device status, signal quality, timestamps, and anomaly flags
- Store raw and processed telemetry
- Support multiple nodes

PRODUCT MODULES

A. ORBIT-MESH CORE
This is the main mesh platform.
- BLE node discovery
- Node pairing
- Live telemetry dashboard
- Message relay
- Node health status
- Offline-first design
- Mesh-style propagation logic
- Stored history and replay

B. HELIO / VLF OBSERVATION
- Real VLF signal ingestion from hardware nodes
- Live signal graph
- Band-energy visualization
- Noise floor tracking
- Peak frequency display
- Event candidate tagging
- No fake “Schumann detected” claims
- Only measure and visualize what the hardware truly measures

C. EARTHSIGN
- Authenticity / provenance layer for media and events
- A signed timestamp + sensor-context record attached to an event capture
- Store metadata for verification
- Do not claim impossible anti-deepfake magic
- Present it as an evidence/provenance system using sensor context, timestamps, and device identity

D. AUET
Acoustic Underground Event Tracker
- Event logging for acoustics / vibration / underground sound observations
- If no real microphone hardware is connected, keep this as a software-ready module with real schema and UI, but no fake sensor readings
- Event timeline, tags, notes, location, and correlation ready

E. HAYATAĞI
This is the biggest retention and daily-use feature.
- Family / classroom / neighborhood groups
- Check-in system
- Daily “I am okay” flow
- Status states: green / yellow / red
- Time-based escalation
- Local and group notification flow
- Offline relay design
- History streaks and safety timeline
- This must be one of the most polished modules in the app

F. ASTRONOMY GAME
- Highly engaging astronomy mini-game
- Beautiful 3D / animated UI
- Real gameplay loops
- Progression, score, streaks, badges
- Must make users return daily
